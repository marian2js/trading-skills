# Catalyst Framework

Use this framework to map events and milestones into a smaller set of catalysts that actually deserve attention.

## What counts as a catalyst

A catalyst is not just a calendar event.

It can be:

- earnings
- macro releases
- central-bank decisions
- product launches
- investor days
- guidance updates
- regulatory or legal milestones
- trial results
- policy announcements
- commodity or supply-chain developments

## What makes a catalyst useful

A useful catalyst map answers:

- what matters
- when it matters
- who it affects
- how it transmits
- what preparation still needs to be done

## Ranking questions

For each catalyst, ask:

- Is it inside the user's real decision window?
- Can it change the thesis, holding period, or risk budget?
- Does it affect one name or a whole cluster?
- Does it overlap with other catalysts?
- Does it require deeper work now?

## Typical transmission paths

- direct issuer effect
- peer sympathy or read-through
- supplier or customer chain
- sector ETF or index effect
- rates, FX, energy, or commodity channel
- sentiment and positioning shift

## Typical mistakes

- listing every event instead of the few that matter
- ignoring shared dependency on the same catalyst
- failing to connect macro events to single-name risk
- treating a catalyst as a forecast
- forgetting that some catalysts matter only because of timeframe

## Boundaries

This framework does not decide:

- whether a trade should be entered
- what order type should be used
- how much size should be allocated
- whether the portfolio is too concentrated

It only organizes what could move the relevant names or exposures.
