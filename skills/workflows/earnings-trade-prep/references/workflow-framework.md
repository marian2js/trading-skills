# Earnings Trade Prep Framework

Use this framework to decide whether an earnings-related trade or hold plan is actually defensible.

## Core question

The question is not only:

- "Is this report important?"

The real questions are:

- Which report matters most?
- What is the actual debate?
- Does the user want a pre-event trade, a hold-through decision, or just better preparation?
- What breaks the plan?

## Typical routing

If the user starts with many names:

- `watchlist-review`
- `catalyst-map`

For the selected name or group:

- `earnings-preview`
- `evidence-gap-check` if the idea is still underdeveloped
- `thesis-validation`

If broader conditions matter:

- `market-regime-analysis`

If the user wants an actual trade plan:

- `risk-reward-sanity-check`
- `execution-plan-check`
- `portfolio-concentration`
- `position-sizing` or `position-management`

## Stop conditions

Stop when:

- the debate is not clear
- the hold-through logic is weak
- the structure depends on unrealistic event behavior
- the portfolio overlap is too high
- the user is trying to force a yes on an avoid case

## Wrapper discipline

The workflow should:

- identify the few checks that matter most
- stop at the first real blocker
- return a clear event decision
- carry forward a compact trade context only when justified

## Boundaries

This workflow does not predict the earnings reaction.

It only decides whether the user has a defensible event-prep or event-trade plan.
